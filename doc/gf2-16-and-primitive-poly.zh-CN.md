# GF(2^16) 的含义，以及如何找到一个域的本原多项式

本文回答两个问题：GF(2^16) 到底是什么；本库用到的多项式
（0x11D 与 EXT_POLY）是怎么"找"出来的——并附用本库自身代码
完成的程序验证。

相关文档：[本原多项式与生成多项式](polynomials.zh-CN.md)、
[README 的算法核心原理](../README.zh-CN.md#算法核心原理)。

---

## GF(2^16) 的含义

**GF = Galois Field（伽罗瓦域，即有限域）**，GF(q) 表示"恰好有 q 个
元素的域"。域论的基本定理：

- 阶为 q 的有限域存在 ⟺ q 是素数的幂（q = pⁿ）；
- **同阶的有限域在同构意义下唯一**——所以 "GF(2^16)" 指的是那个
  本质上唯一的 65536 元域。

构造它有两条路，结果同构：

1. **直接构造**：GF(2) 上的 15 次以下多项式（即 16 个比特位），
   模一个 GF(2) 上的 16 次不可约多项式。元素 = 一个 16 位整数。
2. **塔式构造（本库采用）**：GF((2^8)²)——以 GF(2^8) 为基域再做一次
   二次扩张。元素是 `a·x + b`，系数 a、b 各是一个 GF(2^8) 字节
   （代码中的 `std::array<uint8_t, 2>`），乘法模
   `EXT_POLY = x² + 2x + 128`（`include/rse/galois_16.hpp`）。

选塔式构造纯粹是工程理由：**复用 GF(2^8) 的整套查找表**。
直接构造的话，2^16 的全乘法表要 4 GiB（不可行），log/exp 表虽只要
256 KiB 但缓存不友好；塔式构造让每次双字节乘法分解为 5 次 GF(2^8)
查表乘法 + 一次归约（`galois_16.hpp` 的 `operator*` 与 `reduce_from`）。

对本库的意义：分片总数上限 = 域的阶（编码矩阵每行需要一个互不相同的
域元素），GF(2^8) 封顶 256 个分片，GF(2^16) 提到 65536。

## 直观类比：本原多项式是"钟面上的 12"，不是"数轴的 0 点"

一个常见的误解是把本原多项式类比成数轴的 0 点。0 点是**加法单位元**，
它在域里的对应物是**域元素 0x00**（`Field::zero()`）；本原多项式甚至
不是域的元素（它是 8 次的，域元素最多 7 次）——它是**定义这个域的
规则**，处在更高一层。准确的类比是**钟表算术里的模数**：

- 钟面上 "12 ≡ 0"，这条回绕规则把无限的数轴卷成 12 个刻度的圈；
- GF(2^8) 里 "x⁸ ≡ x⁴+x³+x²+1"（即本原多项式 ≡ 0），这条回绕规则把
  无限的多项式空间卷成恰好 256 个元素的圈。
  `gen_log_table` 里的 `b = (b - 256) ^ 29` 干的就是"指针越过 12 点
  就拨回来"这件事。

**"本原"是钟面之上的又一层性质：步长能走遍全部刻度。**
光"绕回来"还不够（不可约就够了），本原额外保证：以"乘 2"为步长绕圈，
能不重复地踩遍全部 255 个非零刻度。类比：12 刻度的钟面上每次走 5 格
（gcd(5,12)=1）会踩遍所有刻度；每次走 4 格就只能在 {0,4,8} 打转。
下文验证的 AES 0x11B 就是"步长 4"的情形——2 走了 51 步就提前回到
起点，log 表（"每个刻度是第几步踩到的"记录）就建不起来。

**推论：换多项式 = 换一套刻度编号。** 同阶有限域同构唯一，换一个
本原多项式得到的域"本质相同"，但每个字节代表哪个元素的编号方式变了
——就像同一时刻在不同时区的钟面上读数不同。这正是所有 RS 实现必须
约定同一个 0x11D 的原因：编号不同，校验字节就逐位不同，无法互操作。

| 数轴/钟面概念 | GF(2^8) 对应物 |
| --- | --- |
| 数轴的 0 点（加法单位元） | 域元素 0x00 |
| 钟面的 "12"（回绕规则/模数） | 本原多项式 0x11D |
| 步长与圈长互素、走遍全部刻度 | 本原性（2 是乘法群生成元） |
| 换时区（同一时刻不同读数） | 换多项式（同构域、不同字节编号） |

## 回绕规则的数学推导：x⁸ ≡ x⁴+x³+x²+1

上面类比中的"回绕规则"具体就是这条恒等式。它是**多项式的模运算
（同余）**——和"13 点 ≡ 1 点（mod 12）"是同一种数学，
对象从整数换成多项式。

### "≡" 的含义

记本原多项式 p(x) = x⁸+x⁴+x³+x²+1。"≡"读作"模 p(x) 同余"，
即两边相差 p(x) 的倍数：

```
A(x) ≡ B(x)  (mod p(x))    ⟺    A(x) − B(x) 能被 p(x) 整除
```

GF(2^8) 中所有运算都"模 p(x)"进行——正如钟面世界里所有运算都模 12。

### 两步推导

**第一步**：任何数模自身都是 0，p(x) 也不例外：

```
x⁸ + x⁴ + x³ + x² + 1 ≡ 0    (mod p(x))
```

**第二步**：移项，把 x⁸ 留在左边，其余移到右边变号：

```
x⁸ ≡ −(x⁴ + x³ + x² + 1)
```

而系数在 GF(2) 中模 2（1+1=0，故 **−1 = +1**），负号直接消失：

```
x⁸ ≡ x⁴ + x³ + x² + 1    (mod p(x))
```

### 用途：把"溢出"换算回 8 位以内的替换规则

乘法会把次数顶高。每当结果冒出 x⁸ 项（第 9 个比特，即字节溢出），
就用这条恒等式把 x⁸ **原地替换**成 x⁴+x³+x²+1，结果立刻回到 8 位以内。
落到比特层面：

```
溢出位出现  →  丢掉溢出位，再异或 0b0001'1101（= 0x1D = 29）
```

这正是 `gen_log_table` 中那行代码（也解释了代码常量为何只存低 8 位
的 29 而非完整的 0x11D——x⁸ 位总是先被"减掉"）：

```cpp
b <<= 1;                          // 乘以 x，可能冒出第 9 位
if (FIELD_SIZE <= b) {
    b = (b - FIELD_SIZE) ^ 29;    // b-256 丢掉 x⁸ 位，^29 补上替换项
}
```

### 算例（可用本库验证）

| 算式 | 过程 | 结果 |
| --- | --- | --- |
| 2 × 128 | 128 = x⁷，乘 x 得 x⁸ → 替换 → x⁴+x³+x²+1 | **29**（`gf8::mul(2,128)` ✓） |
| 2 × 200 | 200 左移 = 400 溢出 → 丢第 9 位得 144 → 144 ^ 0x1D | **141**（`gf8::mul(2,200)` ✓） |

普通算术里 2×128 = 256 正好"溢出"，被替换规则卷回 29。

**总结**：`x⁸ ≡ x⁴+x³+x²+1` 不是待求解的方程，而是从"p(x) ≡ 0"
移项得到的**替换规则**——它就是这个域的"进位规则"：普通算术满 10
进 1，这个域里"满 x⁸"就拆成 x⁴+x³+x²+1 塞回低位。整个 64 KiB 的
GF(2^8) 乘法表，本质上是"移位 + 这条替换规则"反复应用的结果缓存。

## 如何找到本原多项式

**没有解析公式，本质是"枚举候选 + 程序判定"**，判定分两步。

### 第一步：判不可约

n 次多项式 f 在 GF(2) 上不可约，标准算法是 **Rabin 测试**：验证

```
x^(2ⁿ) ≡ x (mod f)，且对 n 的每个素因子 q：gcd(x^(2^(n/q)) − x, f) = 1
```

原理、复杂度、手算例子与完整实现详见专篇
[rabin-irreducibility-test.zh-CN.md](rabin-irreducibility-test.zh-CN.md)。
低次多项式直接试除所有低次因子即可。**二次多项式更简单：没有根
即不可约**（有限域上的有根判定只需把所有元素代入一遍）。

### 第二步：判本原

不可约只保证商环是域；本原还要求 **x（即元素 2）的阶恰好是 2ⁿ−1**
（乘法群的生成元）。判定方法：对 2ⁿ−1 的每个素因子 p，验证

```
x^((2ⁿ−1)/p) ≢ 1  (mod f)
```

对 GF(2^8)：255 = 3 × 5 × 17，只需检查 x^85、x^51、x^15 都不为 1
——三次幂运算即可完成判定。

### 实践中的三种获取方式

本原多项式并不稀有（n 次的有 φ(2ⁿ−1)/n 个，8 次的有 16 个），
所以实践中：

1. **查文献/沿用惯例**——本库 GF(2^8) 的 `PRIMITIVE_POLYNOMIAL = 29`
   （0x11D）即此。标准表早已枚举完，RS 社区统一用 0x11D 以保证
   互操作（本库与 Backblaze / Klaus Post / Rust 版字节兼容的前提之一）。
   它的本原性还被本库**间接验证**着：`gen_log_table` 循环 255 次，
   若 2 的阶不足 255 就会提前绕圈、对数表出现重复，测试套件中与
   Backblaze 表的逐项比对和域公理测试会立即失败。
2. **工具现搜**——扩域的 `EXT_POLY = x² + 2x + 128` 是原 Rust 作者用
   sage 的 `R.irreducible_element(2, algorithm="first_lexicographic")`
   搜出来的：按字典序枚举候选、做不可约判定、取第一个
   （系数小，归约便宜）。注意它**只需不可约、不必本原**：
   gf16 不用 log/exp 表（求逆走扩展欧几里得算法），不需要生成元。
3. **随机搜 + 判定**——需要非标准参数时的通用做法，因密度高很快命中。

## 程序验证

以下三条均可用本库自身的 GF(2^8) 运算直接验证（完整程序见文末）：

| 验证项 | 方法 | 结果 |
| --- | --- | --- |
| 0x11D 本原 | 2^85、2^51、2^15 是否均 ≠ 1（255 的素因子 3、5、17） | 214、10、38 ✓ |
| EXT_POLY 不可约 | x²+2x+128 在 GF(2^8) 全部 256 个元素中找根 | 0 个根 ✓ |
| 反例：AES 的 0x11B | 不可约但**不本原**——计算 2 在 0x11B 域中的阶 | 阶 = 51（< 255）✗ |

第三条具体演示了"不可约 ≠ 本原"：拿 0x11B 跑 `gen_log_table` 的循环，
51 步后就开始绕圈，对数表会建错——这正是 RS 实现必须选**本原**
多项式的原因。

```cpp
// 用本库自身的 GF(2^8) 运算验证两个多项式的性质
#include <cstdio>
#include <rse/galois_8.hpp>
namespace gf8 = rse::gf8;

int main() {
    // 1) 0x11D 的本原性：255 = 3*5*17，验证 2^(255/p) != 1
    std::printf("2^85 = %d, 2^51 = %d, 2^15 = %d\n",
                gf8::exp(2, 85), gf8::exp(2, 51), gf8::exp(2, 15));

    // 2) EXT_POLY = x^2 + 2x + 128 的不可约性：在 GF(2^8) 中无根
    int roots = 0;
    for (int t = 0; t < 256; ++t) {
        const auto u = static_cast<std::uint8_t>(t);
        if ((gf8::mul(u, u) ^ gf8::mul(2, u) ^ 128) == 0) ++roots;
    }
    std::printf("x^2+2x+128 的根的个数 = %d\n", roots);

    // 3) 反例对照：0x11B 域中 2 的阶（独立小实现，因为本库的表是 0x11D 的）
    auto mul11b = [](unsigned a, unsigned b) {
        unsigned r = 0;
        while (b) { if (b & 1) r ^= a; b >>= 1; a <<= 1; if (a & 0x100) a ^= 0x11B; }
        return r & 0xFF;
    };
    unsigned p = 2; int order = 1;
    while (p != 1) { p = mul11b(p, 2); ++order; }
    std::printf("0x11B 域中 2 的阶 = %d\n", order);
    return 0;
}
```

## 总结

- **GF(2^16)** = 那个（同构唯一的）65536 元有限域；本库用 GF(2^8) 上的
  二次扩张实现，为的是复用字节级查找表；
- **找本原多项式**没有公式，靠"枚举 + 两步判定"（Rabin 判不可约，
  再对 2ⁿ−1 的素因子逐个验幂判本原）；
- 本库的 0x11D 沿用社区标准常量（互操作所需），EXT_POLY 是 sage
  按字典序搜出的第一个不可约二次多项式（只需不可约，不必本原）。
